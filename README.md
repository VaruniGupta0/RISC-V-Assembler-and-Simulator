# Custom RISC-V Assembler & Simulator (RV32I)

A Python-based systems software suite designed to assemble and simulate a core subset of the RISC-V (RV32I) Instruction Set Architecture. This project features a custom two-pass assembler that translates human-readable assembly into machine-level binary, paired with an architectural simulator that executes the binaries while tracking register states, memory allocation, and the Program Counter (PC).



## System Architecture

The project is divided into two primary, decoupled subsystems:

### 1. The Assembler
Translates raw RISC-V assembly instructions into 32-bit machine code.
* **Two-Pass Resolution:** Handles forward and backward label branching seamlessly.
* **Instruction Parsing:** Accurately decodes and encodes opcodes, registers, and immediate values for various instruction formats.
* **Robust Error Handling:** Detects and flags syntax errors, invalid opcodes, out-of-bounds immediates, and undefined labels, outputting precise error logs.

### 2. The Simulator
A virtual machine that executes the binary files generated by the assembler.
* **Architectural State Tracking:** Maintains the state of the 32 general-purpose registers (`x0` to `x31`), keeping `x0` hardwired to zero.
* **Program Counter (PC) Management:** Tracks the execution flow, appropriately handling jumps and conditional branches.
* **Memory Management:** Simulates a virtual memory space to handle load/store operations and instruction fetching.
* **Execution Cycle:** Implements a strict Fetch-Decode-Execute pipeline loop.


## Supported RV32I Instruction Formats

The suite supports the fundamental instruction formats of the base 32-bit integer instruction set:

* **R-Type:** Register-to-register operations (e.g., `add`, `sub`, `and`, `or`, `sll`, `slt`).
* **I-Type:** Register-immediate operations and loads (e.g., `addi`, `lw`. `jalr`).
* **S-Type:** Store operations to memory (e.g., `sw`).
* **B-Type:** Conditional branching (e.g., `beq`, `bne`, `blt`, `bge`).
* **J-Type:** Unconditional jumps (e.g., `jal`).
* **U-Type:** Upper immediate operations (e.g., `lui`, `auipc`).
